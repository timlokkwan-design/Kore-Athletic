## B. 全新版本建議優先修（結構／安全）

### B1. 根目錄大量重複檔（高優先）
- 約 45+ 個 `.py` 同時存在於**根目錄**與 `utils/` / `views/`，內容**不同步**。
- 正式 App 只跑 `app.py` → `utils.*` / `views.*`；根目錄多數是**死碼**。
- 風險：改錯檔以為修好了，實際上線沒變。
- 全新版本建議：刪除或移走根目錄重複 `.py`，單一真相來源只留 `utils/` + `views/`。

### B2. 安全／測試帳殘留（高優先）
- README 明文測試帳密（`ktll`/`170330` 等）。
- 正式／Supabase 路徑仍可能確保測試教練 **`T` / `T`**。
- Cookie 後備密鑰寫死在程式（Secrets 未設時可偽造登入）。
- 密碼最短僅 3 字元；部分腳本硬編碼預設教練密碼。
- Supabase 用 service_role 在前端等同全表讀寫。

### B3. 資料寫入不可靠（高優先）
- Supabase：`清空整表再批次 insert`（併發會互蓋／短暫空表）。
- CSV：整檔覆寫，多人同時操作有 race。
- Session cache 只在「自己寫入」後失效，其他使用者可能看到舊資料。
- 頭像只存本機 `data/avatars/`，Cloud 重部署會丟。

## C. 功能缺口（README 承諾 vs 實際）

| 角色 | 缺口 |
|------|------|
| **學生** | README 有 ACWR／訓練看板，現行學生端無；看不到教練影片分析；忘記密碼只能找教練 |
| **家長** | 無自助註冊／綁定子女流程；功能薄（無細出席、無影片、無比賽報名視圖） |
| **教練** | 「家長溝通」只產生文字摘要，無一鍵 WhatsApp；無系統內家長帳號管理 |
| **訪客** | 僅介紹＋學員註冊 |

其他：
- README 結構過時，引用不存在的 `上線指南.md`、`設定Supabase.md`。
- `PRODUCTION_MODE.txt` 放根目錄不會被讀；正式模式只認 `data/.production`。

## D. Mobile／Streamlit 已知痛點

- 側邊欄預設收合，手機登入依賴 ☰。
- 課表 FullCalendar iframe：觸控／高度／rerun 易不穩。
- 登入／導覽靠 JS cookie + localStorage bridge：Safari／PWA／iframe 下脆弱。
- Sticky 手機導覽依賴 `:has()`，舊瀏覽器可能失效。
